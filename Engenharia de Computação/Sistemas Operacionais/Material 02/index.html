Revisão e Conexão

Na nossa aula inaugural, exploramos como o hardware e o software evoluíram de válvulas para processadores multicore. Entendemos que o SO é o gerente de recursos. Mas agora, como Arquiteto de Sistemas, você deve se perguntar: "Como esse gerente é construído por dentro?".

A Ponte para o Próximo Nível

Não basta saber que o SO gerencia memória. É preciso entender se ele faz isso dentro de um bloco único de código (Monolítico) ou se ele delega essa função para pequenos serviços isolados (Microkernel). Essa decisão de design impacta diretamente na estabilidade de um servidor de missão crítica ou na performance de um console de jogos.

Ponto de Engenharia: O hardware é "burro". Ele apenas executa instruções. O SO é a inteligência que organiza o caos. Nesta aula, abriremos o capô do motor e analisaremos as peças fundamentais da engenharia de software de baixo nível.

Arquiteturas de Kernel: O Coração do SO

O Kernel (núcleo) é a parte do SO que reside permanentemente na memória RAM. Ele tem controle total sobre o sistema. Existem três abordagens principais para desenhá-lo:

1. Kernel Monolítico

Todas as funções do SO (gerência de processos, memória, drivers, rede) rodam em um único espaço de endereçamento de alta autoridade.

Vantagens:
Alta performance (comunicação interna rápida) e maturidade (Ex: Linux, FreeBSD).
Desvantagens:
Baixa tolerância a falhas. Se um driver de vídeo travar, o sistema inteiro pode sofrer um Kernel Panic.

2. Microkernel

O Kernel é reduzido ao mínimo necessário (comunicação entre processos e gerência de interrupções). Funções como drivers e sistemas de arquivos rodam como processos comuns.

Vantagens:
Alta segurança e modularidade. Se um driver falha, basta reiniciá-lo sem desligar o SO (Ex: MINIX, QNX).
Desvantagens:
Perda de performance devido à troca de contexto constante (IPC - Inter-Process Communication).

3. Kernel Híbrido

O equilíbrio moderno. Estruturalmente parece um Microkernel, mas roda a maioria dos componentes no espaço do Kernel para ganhar performance.

Exemplo Principal: Windows NT (Utilizado no Windows 10/11) e MacOS (XNU).

Organização em Camadas e Privilégios

Para um processador, nem todo código é igual. Para proteger o hardware de programas maliciosos ou erros de lógica, utilizamos a separação por Anéis de Proteção (Ring Levels).

USER SPACE (RING 3)
KERNEL SPACE (RING 0)
HARDWARE

Estrutura de privilégios da arquitetura x86/x64.

User Space (Espaço do Usuário)

Onde rodam seus aplicativos (Chrome, VS Code). É um ambiente restrito. Se o programa tentar acessar o disco diretamente, o processador gera uma exceção.

Kernel Space (Espaço do Sistema)

Onde mora o SO. Possui privilégios totais para tocar no hardware. É a área de Ring 0.

Chamadas de Sistema (System Calls)

As System Calls são o "guichê de atendimento". Quando o seu código C++ quer escrever um arquivo, ele chama uma função especial que faz o processador mudar do Ring 3 para o Ring 0, executa a tarefa com segurança e volta para o programa original.

Virtualização: O Hardware como Software

Virtualização é a técnica de criar uma versão virtual (não física) de um recurso, como um servidor, sistema operacional ou rede. Para o Engenheiro de Computação, é a ferramenta suprema para testes e isolamento.

O Hypervisor (VMM)

O Hypervisor é o software que cria e roda as máquinas virtuais. Ele se divide em dois tipos:

Tipo 1: Bare Metal
Nativo

Roda diretamente sobre o hardware. É o SO de si mesmo. Focado em Data Centers.

  • VMware ESXi
  • Microsoft Hyper-V
  • Xen
Tipo 2: Hosted
Hospedado

Roda como um aplicativo dentro de um SO comum (Windows/Linux). Focado em desenvolvedores.

  • Oracle VirtualBox
  • VMware Workstation
  • QEMU

Hardware Assisted Virtualization

Antigamente, virtualizar era lento (o software emulava cada instrução). Processadores modernos possuem instruções específicas (Intel VT-x e AMD-V) que permitem que a VM execute código quase na velocidade do hardware real.

Recurso Valor Mínimo Justificativa

Dica de Lab: Certifique-se de que a virtualização está ativada na BIOS/UEFI do seu computador real. Se você receber um erro "VT-x is disabled", precisará entrar no setup da placa-mãe ao ligar o PC.
