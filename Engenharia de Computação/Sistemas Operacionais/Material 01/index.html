Definição: O "Mestre de Obras" do Computador

Um Sistema Operacional (SO) é o software mais importante que roda em um computador. Ele gerencia a memória e os processos, bem como todo o software e hardware. Sem ele, você teria que escrever código em binário para cada pequena ação (como salvar um arquivo).

As 4 Funções Fundamentais

Gerenciamento de Processos

O SO decide qual programa usa a CPU agora, por quanto tempo e com qual prioridade. Sem isso, um programa travado congelaria toda a máquina.

Gerenciamento de Memória

Alocação de espaços na RAM para que os programas não "atropelem" os dados uns dos outros. Inclui a poderosa Memória Virtual.

Sistemas de Arquivos

Organização lógica dos dados. O SO transforma trilhas e setores físicos do disco em "pastas" e "arquivos" compreensíveis.

Gerenciamento de E/S (Entrada/Saída)

Comunicação com dispositivos externos através de Drivers. O SO padroniza como o computador conversa com o teclado ou a impressora.

O Túnel do Tempo dos Bits

A evolução dos Sistemas Operacionais caminha de mãos dadas com a evolução do Hardware. Vamos analisar as quatro gerações clássicas:

1945 - 1955

1ª Geração: Tubos de Vácuo (Válvulas)

Cenário: Máquinas colossais que ocupavam salas inteiras. Não havia "Sistema Operacional".

  • Programação via painéis de plugues e chaves físicas.
  • Linguagem de máquina absoluta.
  • Apenas uma pessoa usava por vez.
"Se um tubo de vácuo queimasse (e eles queimavam toda hora), o processamento parava."
1955 - 1965

2ª Geração: Transistores e Sistemas Batch

Cenário: O surgimento dos computadores mainframe (ex: IBM 7094).

  • Sistemas em Lote (Batch): Programas gravados em cartões perfurados. O SO lia uma pilha de cartões e executava um por um.
  • Surgimento do programador vs operador.
  • Linguagens: FORTRAN e Assembly.
1965 - 1980

3ª Geração: Circuitos Integrados (ICs)

Cenário: A era da Multiprogramação e do Compartilhamento de Tempo.

  • Multiprogramação: Vários jobs na memória ao mesmo tempo. Enquanto um faz I/O, a CPU processa outro.
  • Time-Sharing: Usuários em terminais remotos compartilhando a mesma CPU.
  • Nascimento do UNIX.
1980 - Presente

4ª Geração: Computadores Pessoais (PC)

Cenário: O computador em cima da mesa. Interface Gráfica (GUI) vira padrão.

  • Microsoft MS-DOS e Windows.
  • MacOS e interfaces WIMP (Windows, Icons, Menus, Pointers).
  • Sistemas de Rede e Internet integrados.
Ponto de Reflexão: Atualmente vivemos a era dos sistemas Multicore, Móveis e Virtualizados. Nesta disciplina, focaremos em como gerenciar essa enorme complexidade de hardware atual (64 núcleos, terabytes de memória) mantendo a estabilidade.

CLI vs GUI: A Interface com o Mundo

Um Sistema Operacional pode oferecer diferentes formas de interação. Para o usuário comum, a interface é o sistema. Para o engenheiro, a interface é apenas uma ferramenta.

Desafio de Engenharia: CLI ou GUI?

Imagine que você precisa criar 1.000 pastas numeradas de 1 a 1.000. No GUI, você teria que clicar e renomear 1.000 vezes. Qual comando terminal você usaria no Linux para resolver isso em 2 segundos?

mkdir pasta_{1..1000}

Percebe a força da CLI para um Engenheiro? É sobre eficiência em escala.
