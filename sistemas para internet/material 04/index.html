01. A Interface como Experiência

🎨

UI (User Interface)

É a camada visual. Cores, tipografia, botões e espaçamento. É o que o usuário toca e vê.

🧠

UX (User Experience)

É a jornada. O fluxo, a facilidade de uso e a lógica. É o que o usuário sente ao navegar.

"Uma UI bonita não salva uma UX confusa, mas uma UX ruim estraga qualquer design."

Por que UI/UX no TCC?

A interface é o primeiro ponto de avaliação da Banca Examinadora. Antes mesmo de verem seu banco de dados, eles verão sua tela de login.

Critério de Avaliação:

Um sistema com interface desleixada transmite a sensação de que o código também é desorganizado. Um design profissional eleva sua nota AV1 e AV3 instantaneamente.

02. Semântica e Acessibilidade

HTML Semântico não é apenas "organização", é dar significado ao conteúdo para Google (SEO) e Leitores de Tela.

Errado (Divite)

<div class="titulo">Meu TCC</div>
<div class="menu">...</div>

Correto (Semântico)

<header>
  <h1>Meu TCC</h1>
  <nav>...</nav>
</header>

Diretrizes de Acessibilidade (WCAG)

1

Contraste de Cores

Garanta que textos tenham contraste suficiente contra o fundo (AA: 4.5:1).

2

Navegação via Teclado

Todos os botões e links devem ser acessíveis usando a tecla [TAB].

03. Prática: Codificação das Telas

Nesta fase, você deve escolher sua "pilha" de estilo. Recomendação de mercado:

Tailwind CSS (Recomendado) Bootstrap 5 Styled Components

Dica: Frameworks aceleram o desenvolvimento e garantem consistência visual.

Tela Crítica 1: Login e Cadastro

Requisitos UI:

  • ● Mensagens de erro claras (Input vermelho).
  • ● Botão de ação principal (Primary Action) destacado.
  • ● Layout centralizado e minimalista.

Tela Crítica 2: Dashboard/Listagem

Pense na hierarquia: o que é mais importante deve ser maior e mais escuro.

Tela Crítica 3: Formulários (CRUD)

Dica de UX: Use máscaras de entrada (Data, CPF) para evitar erros do usuário.

Guia Rápido: Flexbox

Essencial para alinhar elementos em uma dimensão (Linha ou Coluna).

.container {
  display: flex;
  justify-content: center; /* Alinha horizontal */
  align-items: center; /* Alinha vertical */
  gap: 1rem; /* Espaço entre itens */
}

Guia Rápido: CSS Grid

Perfeito para layouts complexos de duas dimensões (Painéis/Dashboards).

.grid-layout {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

4. Workshop PBL: Prototipagem

Desafio de Design

Traduza os requisitos do seu TCC em uma Sidebar (Barra Lateral) funcional que contenha pelo menos 4 links e a foto do usuário logado.

Critério de Sucesso:

A navegação deve ser consistente em todas as telas do sistema.

Exemplo de Sidebar Semântica

<aside class="sidebar">
  <nav>
    <ul>
      <li><a href="/home">Home</a></li>
      <li><a href="/relatorios">Relatórios</a></li>
    </ul>
  </nav>
</aside>

Usamos <aside> porque o conteúdo da barra lateral é "tangencial" ao conteúdo principal.

05. Auditoria de Acessibilidade

Abra o Chrome, aperte F12 e vá na aba Lighthouse.

Checklist de Auditoria:

Quiz Semântico

"Qual tag é mais adequada para o título principal da página de perfil do usuário?"

Box: Usabilidade Mobile

Responsividade no TCC

Não desenvolva pensando apenas em monitores de 27 polegadas. A banca pode testar seu sistema em um notebook de 13 ou até num tablet.

Regra de Ouro: Use unidades relativas como `%`, `rem` ou `vw` em vez de pixels fixos. No Tailwind, abuse dos prefixos `sm:`, `md:` e `lg:`.

Psicologia das Cores

A cor do seu TCC comunica sem palavras:

AZUL: Confiança, Segurança, Institucional
VERDE: Saúde, Crescimento, Sucesso
VERMELHO: Erro, Atenção, Urgência
AMARELO: Alerta, Otimismo, Cuidado

Tipografia e Leitura

Para aplicações web de gestão (TCCs comuns), utilize fontes sem serifa (Sans Serif) para melhor leitura em telas.

Recomendações:

  • Inter (A favorita dos designers SaaS)
  • Roboto (Padrão Android)
  • Poppins (Moderna e geométrica)

White Space (Espaço em Branco)

Não tenha medo de deixar espaços vazios. O espaço em branco reduz a carga cognitiva e guia o olhar do usuário.
